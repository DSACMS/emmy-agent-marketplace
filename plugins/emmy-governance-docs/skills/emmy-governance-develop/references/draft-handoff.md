# Handoff To Emmy Governance Doc Draft

Use this reference before telling the human that a concept is ready for
`emmy-governance-doc-draft`.

## Minimum Handoff Inputs

The concept packet should contain enough information for the draft skill to
resolve or explicitly mark these fields:

- document title or working title;
- domain;
- purpose or obligation;
- applies-to scope;
- does-not-apply exclusions;
- current process description;
- process roles and responsibilities;
- tools and systems, or explicit statement that none are required;
- Driver role or team;
- one Approver role;
- Contributors roles or teams;
- Informed roles or teams;
- review cadence;
- requirement citation when an external obligation applies;
- Control ID and Control Family when ATO-related.

## Handoff Language

If ready, say:

```text
This concept is ready for $emmy-governance-doc-draft. Use the concept packet as
the source and do not invent missing process decisions.
```

If almost ready, name the missing facts and ask for them before invoking the
draft skill.

If not ready, keep developing the concept. Do not route to the draft skill just
because the human wants a polished document.

## Boundary

This skill may recommend handoff, but it must not draft or create the final
governance document. The draft skill owns document structure, Confluence page
preview, approval gate, creation, labeling, and post-write verification.
