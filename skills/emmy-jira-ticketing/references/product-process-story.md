# Product And Process Stories

Use this reference for product, state-facing, pilot-ready, policy/process,
roadmap, or State TA collaboration work.

## Field Pattern

- Issue type: often `Story`; use `New Feature` or `Improvement` only when the
  human asks or nearby matching tickets strongly use those types.
- Priority: often `High` when the work is pilot-ready, state-facing, roadmap
  critical, or needed for immediate state collaboration.
- Labels: product/process stories may have no labels. Do not force `engineering`
  unless implementation is part of the ticket.
- Epic Link: leave unset unless a matching product epic is clear.

## Description Shape

Common structure:

- A user-story paragraph
- `Acceptance Criteria`
- Optional business metadata such as Squad, Feature Area, Roadmap Phase,
  Priority, and Notes

Acceptance criteria can describe process outputs, state sign-off, approved
messaging, documentation, or change-routing expectations. The output does not
need to be a software artifact.
